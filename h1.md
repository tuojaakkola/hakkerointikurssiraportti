# Scope First, Controls Later

## a)

Soveltamisalaan kuuluvat seuraavat osa-alueet: (In-Scope)

#### Kotiverkon perusinfrastruktuuri
* Internet-yhteys (taloyhtiön tarjoama Elisa-yhteys)
* Verkkolaitteet: kytkin/jakorasia (TP-Link TL-SG1005D), Ei reititintä / wifiä.
* Kotiverkon sisäiset kaapeloinnit

#### Kurssiharjoituksiin käytettävät laitteet
* Pöytäkone (Windows)
* Läppäri (Windows), pääasiallinen opiskelulaite, joka sisältää virtuaalikoneet.
* Virtuaalikoneet (Oracle VirtualBox, Linux Ubuntu):
  * Master image (NAT, pääsy internetiin)
  * Linked clone (NAT, pääsy internetiin)
  * Linked clone (Internal Network, ei internet-yhteyttä)
* Oma älypuhelin (iPhone 13), jota käytetään jakamaan pääsääntöisesti internet yhteys läppärille sekä Microsoft Authenticator-tunnistautumiseen.

#### Tiedot
* Identiteetitiedot: Microsoft Authenticator
* Lähdekooditiedot: GitHub repositoriot
* Opiskeludokumentaatiot: Pilvitallennuspalveluissa säilytettävät opiskeludokumentit (Microsoft 365 / OneDrive, iCloud)

### Ulosrajaukset: (Out-of-scope)

* **Älytelevisio:** Ei sisällä opiskeluun liittyvää tietoa eikä ole kriittinen kurssin näkökulmasta. On myös kytketty kiinteään verkkoo joten se on eriytetty kurssikäytössä olevasta läppäristä ja virtuaalikoneista.
* **Taloyhtiön internet operaattorin verkon puolella:** Ei ole minun hallinnassani ja perustuu palveluntarjoajan vastuuseen.
* **Mobiilioperaattorin verkko:** Ei ole minun hallinnassani ja perustuu palveluntarjoajan vastuuseen.

Rajaukset perustuvat omistajuuteen, hallittavuuteen sekä siihen, että rajatut kohteet eivät ole merkityksellisiä kurssin tietoturvatavoitteiden kannalta.

### Keskeiset rajapinnat

* **Verkkorajapinnat:** Iphonen (Hotspot), joka toimii palomuurina läppärin ja internetin välillä. PC:n kiinteä verkkoyhteys joka yhdistyy kytkimen kautta taloyhtiön verkkoon. PC toimii siis eri verkossa ja on eriytetty opiskelukokoonpanosta.
* **Sisäiset rajapinnat:** Windows-isäntäkoneen ja Linux pohjaisten virtuaalikoneiden välinen liikenne. Virtuaalikoneet toimivat omassa "hiekkalaatikossaan" ja rajapinta sieltä host koneelle on eristetty.
* **Yksisuuntainen tiedonsiirto:** Oikean ja suljetun virtuaalikoneen välillä toimii rajapinnaksi luodun kansion avulla(Read Only). Voin kopioida työkaluja Windowsista virtuaalikoneelle, mutta virtuaalikone ei voi kirjoittaa tai muuttaa mitään läppärin tiedostoja.
* **Pilvipalveluiden toimittajat:** Github, Microsoft 365

### Mitä näyttöä voisin esittää?

* **Verkkoeristys:** Kuvakaappaus VirtualBoxissa olevien virtuaalikoneiden verkkoasetuksista
* **Laitelista:** Kuvakaappaus kaikista luoduista virtuaalikoneista ja niiden tilannevedoksista (Snapshots).
* **Tiedonsiirtorajoitukset:** Kuvakaappaus jaetun kansion asetuksista joissa näkyy "Read Only" valittuna.
